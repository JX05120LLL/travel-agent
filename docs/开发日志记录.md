## 2026-04-15 | Phase 2 & Phase 3 收尾

### 今天做了什么

**Phase 2（Tool Use）收尾**
- 修复了 `test_tool_use.py` 第二轮 LLM 调用的 bug：第二轮误用了 `llm_with_tools`，导致 LLM 拿到工具结果后不直接回答，而是继续尝试调工具。改为 `llm.invoke()` 后正常
- 学习了 Tool Use 核心概念：`@tool` 装饰器的 docstring 是给 LLM 读的（不是给开发者看的），LLM 靠它判断何时调用工具；`bind_tools` 把工具 schema 发给 LLM；实际执行工具的是 Python 代码，LLM 只负责决策

**Phase 3（LangGraph Agent）完成**
- 创建了 `agent/state.py`、`agent/prompts.py`、`agent/graph.py`、`test_agent.py`
- 修复了 LangGraph 节点命名 bug：`tools_condition` 内置函数固定返回字符串 `"tools"`，节点必须命名为 `"tools"` 否则报 `ValueError`
- 理解了 LangGraph 五个核心概念：State（共享上下文）、Node（处理函数）、Edge（固定路径）、Conditional Edge（条件路由）、Graph 循环（tools → llm_node 形成循环，LangGraph 自动处理）

### 遇到的问题 & 解决过程

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| LLM 第二轮不给完整回答 | 第二轮用了 `llm_with_tools`，LLM 想再调工具 | 改为 `llm.invoke()`，强制直接回答 |
| LangGraph 启动报 ValueError | 节点命名为 `"tool_node"`，但 `tools_condition` 返回 `"tools"` | 节点重命名为 `"tools"` |
| 天气 API 一直 404 | 旧域名（`geoapi.qweather.com`、`devapi.qweather.com`）从 2026 年起逐步废弃，新账号无权限 | 去控制台 Settings 获取个人专属 API Host，写入 `.env` |
| 天气 API 换了 Host 还是 404 | GEO 接口路径写错：`/v2/city/lookup` → 正确是 `/geo/v2/city/lookup` | 查阅官方文档，修正路径；同时改用 Header 方式传 Key（`X-QW-Api-Key`） |
| 每次启动第一次调用偶发失败 | DeepSeek API 冷连接偶尔超时，没有重试机制 | 添加 `invoke_with_retry()` 函数，失败自动重试一次 |
